# Deadlock SteamDB Tracker Bot 🤖🎮

Telegram-бот для автоматичного відстеження оновлень гри **Deadlock** на SteamDB та надсилання сповіщень у ваш Telegram-канал при зміні `changenumber`.

Цей бот спеціально оптимізований для стабільної та надійної роботи 24/7 на Linux-серверах (Ubuntu/Debian) та використовує швидкий, офіційний протокол отримання даних через SteamCMD API, що повністю запобігає блокуванням з боку захисту Cloudflare на SteamDB.

---

## 📋 Моніторинг AppID

Бот відстежує два ключових AppID, пов'язаних із Deadlock:
1. **Deadlock** (Основна версія): `1422450`
2. **Deadlock Experimental**: `3488080`

---

## 🛠️ Як працює трекер

1. Бот надсилає запит до джерела даних Steam PICS через API раз на $N$ секунд (налаштовується в `.env`).
2. Отримує поточний `changenumber` (глобальний номер змін Steam) для кожного AppID.
3. Порівнює отриманий номер із записаним у файлі `last_known.json`.
4. Якщо номер змінився (відбулося оновлення гілок, релізів або налаштувань гри), бот надсилає стильне HTML-повідомлення в Telegram-канал і оновлює локальний стан.
5. Час оновлення виводиться одночасно за київським часом та часом у Сієтлі (тихоокеанський час PT, що відповідає відображенню на SteamDB).

## 🚀 Швидкий запуск

### Крок 1. Створення Telegram-бота та каналу

1. **Створення бота**:
   - Перейдіть у Telegram та знайдіть офіційного бота [@BotFather](https://t.me/BotFather).
   - Надішліть команду `/newbot`.
   - Введіть назву бота (наприклад, `Deadlock Tracker Bot`) та його юзернейм (має закінчуватися на `_bot`, наприклад, `DeadlockUpdateTrackerBot`).
   - Скопіюйте отриманий токен (виглядатиме як `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`). **Нікому не показуйте цей токен!**

2. **Налаштування каналу**:
   - Створіть новий публічний або приватний Telegram-канал (або використовуйте існуючий).
   - Додайте вашого створеного бота до каналу як **Адміністратора**.
   - Переконайтеся, що бот має дозвіл **Публікувати повідомлення** (Post Messages).

3. **Отримання Chat ID**:
   - Для **публічного каналу**: ID — це його юзернейм із символом `@` (наприклад, `@my_deadlock_channel`).
   - Для **приватного каналу**: ви можете переслати будь-який пост із каналу боту [@ShowJsonBot](https://t.me/ShowJsonBot) або [@userinfobot](https://t.me/userinfobot) для отримання ID, який зазвичай починається з `-100` (наприклад, `-1001234567890`).

---

### Крок 2. Встановлення та налаштування на сервері (Linux)

Рекомендується запускати бота у віртуальному середовищі Python.

1. **Клонування проєкту (або перенесення коду)**:
   ```bash
   cd /opt
   # Перемістіть директорію deadlock-tracker сюди або клонуйте її
   git clone <посилання_на_репозиторій> deadlock-tracker
   cd deadlock-tracker
   ```

2. **Встановлення Python та Git**:
   ```bash
   sudo apt update
   sudo apt install python3 git -y
   ```

3. **Налаштування конфігурації**:
   Скопіюйте приклад конфігураційного файлу та відредагуйте його:
   ```bash
   cp .env.example .env
   nano .env
   ```
   Внесіть ваші дані у файл `.env`:
   ```env
   TELEGRAM_BOT_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ
   TELEGRAM_CHAT_ID=@ім_я_вашого_каналу
   CHECK_INTERVAL=90
   API_TIMEOUT=15
   ```
   *Рекомендація:* Інтервал перевірки `CHECK_INTERVAL` краще встановити в межах **60–120 секунд**. Це забезпечить високу оперативність та убезпечить від лімітів на запити.

4. **Тестування налаштувань Telegram**:
   Ви можете швидко перевірити правильність налаштування токена бота, chat ID та його доступу до каналу/групи за допомогою тестового режиму:
   ```bash
   python bot.py --test
   ```
   Якщо все налаштовано вірно, бот надішле тестове повідомлення у ваш канал/групу і завершить роботу. Якщо є помилки (невірний токен, ID або відсутність прав адміна), бот виведе детальний опис проблеми.

---

### Крок 3. Налаштування автозапуску через systemd (Linux 24/7)

Щоб бот працював безперервно у фоновому режимі та автоматично перезапускався у разі перезавантаження сервера, створіть системну службу.

1. Скопіюйте шаблон сервісного файлу у каталог systemd:
   ```bash
   sudo cp systemd/deadlock-tracker.service.example /etc/systemd/system/deadlock-tracker.service
   ```

2. Якщо ви розмістили бота не в `/opt/deadlock-tracker` або хочете запустити його від імені іншого користувача (не `root`), відредагуйте файл служби:
   ```bash
   sudo nano /etc/systemd/system/deadlock-tracker.service
   ```

3. Перезавантажте конфігурацію systemd та запустіть сервіс:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable deadlock-tracker
   sudo systemctl start deadlock-tracker
   ```

4. Перевірка статусу служби:
   ```bash
   sudo systemctl status deadlock-tracker
   ```

5. Перегляд логів роботи в реальному часі:
   ```bash
   sudo journalctl -u deadlock-tracker -f
   # Або у файлі логу бота (якщо увімкнено логування у файл):
   tail -f bot.log
   ```

## 📂 Структура проєкту

```text
deadlock-tracker/
├── bot.py                  # Основний код логіки бота (не потребує встановлення бібліотек!)
├── .env.example            # Шаблон конфігураційного файлу
├── last_known.json         # Локальний файл для збереження стану (changenumber)
├── README.md               # Ця інструкція
├── test_bot.py             # Тестовий скрипт для перевірки працездатності
└── systemd/
    └── deadlock-tracker.service.example  # Шаблон для служби systemd
```

## ⚙️ Робота на Windows (для локальних тестів)
Ви також можете запустити бота на ОС Windows:
1. Встановіть Python із офіційного сайту.
2. Створіть файл `.env` із шаблону `.env.example` та пропишіть туди токен та Chat ID (або пропишіть їх безпосередньо всередині `bot.py`).
3. (Опціонально) Перевірте підключення до Telegram:
   ```powershell
   py bot.py --test
   ```
4. Запустіть скрипт у режимі постійного моніторингу:
   ```powershell
   py bot.py
   ```
